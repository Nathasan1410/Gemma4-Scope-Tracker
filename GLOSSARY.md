# Glossary

- **LiteRT-LM**: LiteRT-based local model runtime for mobile (primary special track target).
- **Cactus**: multi-model routing concept/tooling (skip unless real routing exists).
- **llama.cpp**: lightweight C/C++ inference runtime; roadmap path for Linux/Raspberry Pi / host node.
- **Ollama**: desktop model runner; non-core, useful for local testing only.
- **Unsloth**: training/fine-tuning tooling; optional experiment track.
- **LoRA**: Low-Rank Adaptation; small adapter weights applied to a base model.
- **QLoRA**: quantized LoRA fine-tuning approach.
- **SLM**: small language model.
- **GGUF**: common file format used by llama.cpp ecosystem for quantized models.
- **Quantization (Q2/Q3/Q4)**: reducing weight precision/size; higher Q (e.g., Q4) usually preserves quality better than lower Q (e.g., Q2).
- **Survival pack**: offline knowledge bundle (SOPs, checklists, region notes, sources/labels).
- **Model pack**: model runtime package for on-device inference (weights + tokenizer + config).
- **Host mode**: Survival Node direction where one device hosts local access (LAN/hotspot/local web host).
